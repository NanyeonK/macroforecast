macrocast.execution.execute_sweep(*, plan, output_root, local_raw_source=None, execution_route='comparison_sweep', extra_provenance=None)[source]

Execute every variant of plan under output_root.

Failure handling is driven by the parent recipe’s failure_policy meta axis (Layer 0, section 0.4). Supported policies and sweep-level behaviour:

  • fail_fast - the first variant failure re-raises (abort).

  • skip_failed_cell / skip_failed_model / save_partial_results - record the failure in the study manifest and continue.

  • warn_only - same as the continue set above, plus emit a RuntimeWarning per failed variant.

Recipes that do not specify failure_policy default to fail_fast.

Parameters:

  • plan (SweepPlan) – A SweepPlan produced by macrocast.compile_sweep_plan().

  • output_root (str | Path) – Directory that will contain variants/<vid>/ per variant, the shared FRED cache at .raw_cache_shared, and the aggregate study_manifest.json.

  • local_raw_source (str | Path | None) – Passed through to execute_recipe() so tests can use fixture CSVs.

  • execution_route (str) – Recorded on the study manifest. Defaults to comparison_sweep.

  • extra_provenance (dict[str, Any] | None) – Optional dict merged into each variant’s provenance_payload on top of variant_id / study_id.

Returns:

A SweepResult summarising per-variant outcomes.

Return type:

SweepResult
