# Macrocast Full Option-Universe Implementation Roadmap

Goal
- Turn the current grammar-first package into a package that can eventually execute the full option universe explicitly represented in registry/path space.
- Preserve one path = one fully specified forecasting study.
- Preserve the split between representable choice space and current executable support while systematically promoting remaining registry-only/planned choices into operational support.

## 2. Phase roadmap

### Phase A — Feature-builder widening
Status
- mostly complete for fixed single-run routes

### Phase B — Train-only preprocessing widening
Status
- partially complete

What is complete
- explicit `raw_only` remains operational
- train-only raw-panel path with `x_missing_policy = em_impute` and `scaling_policy = standard` is operational
- train-only raw-panel path with `x_missing_policy = em_impute` and `scaling_policy = robust` is operational

### Phase C — Framework widening
Status
- complete for first operational slice

### Phase D — Benchmark-family widening
Purpose
- continue adding benchmark executors independently of model executors

Status
- first custom benchmark bridge complete

What is complete
- `benchmark_family = custom_benchmark` is operational through a plugin-ready local Python callable bridge
- compiler preserves and validates `benchmark_config.plugin_path` plus `benchmark_config.callable_name`
- runtime loads the callable, executes it as the benchmark forecaster, and preserves the benchmark contract in manifest provenance

### Phase E — Evaluation and statistical tests
Status
- partially complete

What is complete
- `comparison_summary.json` is written for every successful run as the baseline comparison-layer artifact
- `stat_test = dm` is operational
- `stat_test = cw` is operational
- runtime writes `stat_test_dm.json`
- runtime writes `stat_test_cw.json`
- manifest records `comparison_file`, `stat_test_spec`, and `stat_test_file`

### Phase F — Importance layer
Status
- partially complete

What is complete
- `importance_method = minimal_importance` is operational
- runtime writes `importance_minimal.json`
- manifest records `importance_spec` and `importance_file`
- first operational support exists for:
  - non-AR linear families: `ridge`, `lasso`
  - one tree family: `randomforest`

What remains
- broader minimal-importance coverage across more model families
- richer importance semantics for lagged-target routes
- `shap`

### Phase G — Data/task widening
Purpose
- widen execution surface beyond revised single-target studies

Status
- real-time explicit-vintage slice complete
- first multi-target point-forecast slice complete
- minimal wrapper handoff contract defined for wider wrapper-owned families
- fixed-vs-sweep / tree-context provenance slice complete

What is complete
- `info_set = real_time` is operational for single-target studies when `leaf_config.data_vintage` is supplied explicitly
- compiler fails closed if real-time is requested without `data_vintage`
- runtime passes explicit vintage requests into raw loaders so raw metadata and artifact provenance switch to vintage mode
- `task = multi_target_point_forecast` is operational for the first narrow shared-environment slice
- compiler fails closed if multi-target is requested without explicit `leaf_config.targets`
- runtime writes one shared run directory with multi-target `predictions.csv`, `metrics.json`, and `comparison_summary.json`
- `study_mode = orchestrated_bundle_study` now emits deterministic `wrapper_handoff` payloads for `multi_target_separate_runs`, `benchmark_suite`, and `ablation_study`
- wrapper-owned studies remain representable-but-not-executable; handoff is explicit and fail-closed
- compiler and runtime manifests now preserve deterministic `tree_context` provenance showing Stage 0 fixed design, varying design, and path-level fixed/sweep/conditional grouping
