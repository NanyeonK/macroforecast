# Epic 3 / Stage 2 Implementation Plan

Goal: complete Stage 2 preprocessing governance and operational widening for Issue 3-1 through 3-9 without overstating runtime support.

Architecture:
- Add missing governance axes under `macrocast/registry/preprocessing/`.
- Expand preprocessing vocabulary in registry + preprocessing contract together.
- Keep compiler/execution honest by threading new governance fields into `PreprocessContract`, manifest, and execution checks.
- Operationalize only preprocessing paths that runtime truly supports now: selected imputation, outlier handling, scaling, PCA/static-factor reduction, and simple feature selection in train-only raw-panel workflows.

Execution steps:
1. Add registry files for `representation_policy`, `axis_role`, `tcode_application_scope`, and remaining Stage 2 auxiliary axes from Issue 3-9.
2. Expand existing preprocessing registries (`x_missing_policy`, `x_outlier_policy`, `scaling_policy`, `dimensionality_reduction_policy`, `feature_selection_policy`) to planned/operational Stage 2 vocabulary.
3. Extend `PreprocessContract` and preprocessing builder/governance logic for new axes and defaults.
4. Update compiler `_build_preprocess_contract()` and manifest serialization to require and record new governance axes while remaining backward compatible via defaults.
5. Extend execution preprocessing application for supported train-only raw-panel paths: mean/median impute, ffill/interpolate, winsorize/iqr/zscore clipping, minmax scaling, PCA/static_factor, correlation_filter/lasso_select.
6. Add explicit incompatibility checks for unsupported combinations instead of silent fallback.
7. Add/expand tests in `tests/test_preprocess_contract.py`, `tests/test_compiler.py`, `tests/test_execution_pipeline.py`, and registry loader coverage.
8. Update docs and `plans/implementation-issues.md` to match actual Stage 2 status.
9. Run focused + broad pytest, review diff, commit.

Validation:
- Focused: `python3 -m pytest tests/test_preprocess_contract.py tests/test_compiler.py tests/test_execution_pipeline.py tests/test_registry_loader.py -q`
- Broad: `python3 -m pytest tests/test_stage0.py tests/test_stage0_completion.py tests/test_raw.py tests/test_raw_adapters.py tests/test_raw_hardened.py tests/test_raw_sd.py tests/test_recipe_execution.py tests/test_preprocess_contract.py tests/test_execution_pipeline.py tests/test_compiler.py tests/test_start.py tests/test_registry_loader.py -q`
