# WP-V3 — Monte Carlo size/power results (`tests/mc/`)

Executed on server1 (48-core, BLAS pinned to 1 thread), branch
`test/mc-size-power-v3`, package imported from the worktree
(`~/project/mf-v3`). Full suite: `pytest tests/mc/ -m mc` →
**14 passed, 7 xfailed (all `strict=True`), 0 failed** in **5m20s** wall
clock — comfortably inside the 20–30 min budget.

Why this WP exists: the parity harnesses (V1 R-parity, V2 model anchors)
prove macroforecast MATCHES a reference implementation on fixed fixtures;
this harness proves the STATISTICAL BEHAVIOR of the inference-critical
forecast-comparison tests is right. A test whose rejection rate under a
true null is far from its nominal α is broken for referees even when it
matches R bit-for-bit — MC size checks catch distribution/HAC/df errors
that shared conventions hide by construction.

Methodology, fixed before looking at any result and never widened after:

- Master seed `20260705`; per-replication RNGs spawned via
  `numpy.random.SeedSequence(master, spawn_key=(salt,)).spawn(R)` with a
  unique integer `salt` per design cell — no stream reuse anywhere.
- Every rejection rate is reported with a **Clopper–Pearson exact-binomial
  99% CI**; PASS means the nominal α (or coverage target) lies inside that
  CI. R per cell chosen so the band is decisive at ~1–3% width
  (R=3000–5000 for cheap analytic tests, R=1000–1500 for bootstrap tests).
- A distortion is reported as a FINDING with `xfail(strict=True)` +
  diagnosis, never absorbed by loosening the band. Every xfail below was
  re-confirmed at a larger R (or across a design sweep) before being
  classified.
- Raw machine-written results: `tests/mc/_mc_raw_results.json`
  (regenerated by every `-m mc` session; the table below is transcribed
  from that file, not from terminal scrollback).

## Findings (each re-confirmed before classification; all shipped as `xfail(strict=True)`)

### Finding 1 — `dm_test` oversized at n=50, h=4 (both corrections; HLN helps but does not fix)

Re-confirmed at R=10,000: hln .074 @α=.05 (CI99 [.067,.081]) and .132
@α=.10; none .089 @α=.05 and .154 @α=.10. **Diagnosis: variance-estimator
small-sample bias, not a df or correction-formula defect.** At n=50, h=4
the acf-kernel LRV uses 3 autocovariance lags over only ~12–13 effective
non-overlapping blocks — too few for the t(n−1) reference. The identical
kernel/correction code path at n=200, h=4 is correctly sized in the same
suite, and the HLN-vs-none direction test PASSES (HLN strictly closer to
nominal at both α), exactly the behavior HLN (1997) documents (their
correction improves but does not fully repair size at very small n/h
ratios). This is a *documented statistical limitation shared with
forecast::dm.test itself* (the formula is parity-matched to R in WP-V1),
not an implementation bug — but referees using h=4 with ~50 OOS points
should know rejection at the margin is unreliable.

### Finding 2 — `giacomini_white_test` genuinely oversized for h>1; does NOT vanish asymptotically

The worst distortion in this WP. At h=4: .124 @α=.05 (n=50) and .089
(n=200) — and a follow-up sweep shows it is NOT small-n noise:

| n | 200 | 500 | 1000 | 2000 | 5000 |
|---|---|---|---|---|---|
| rate @α=.05, h=4 | .082 | .091 | .092 | .088 | .072 |

At n=5000 the statistic's mean is 2.29 vs the chi²(2) reference mean of
2.00. A second sweep at fixed n=500 over h=1..4 shows the distortion is
**monotone in the number of Newey-West HAC lags (h−1)**: h=1 (0 lags) is
correctly calibrated (mean stat 1.95, rate .045); h=2/3/4 give mean stat
2.27/2.45/2.57 and rate .070/.079/.084. **Diagnosis: the HAC-covariance
Wald construction in `giacomini_white_test` (Bartlett weights, h−1 lags,
no small-sample correction) inflates the statistic whenever h>1 —
qualitatively the same mechanism the HLN correction exists to fix for
`dm_test`, but GW here has no analogous correction.** Since the
brief's own naming-trap note applies (this is the REAL Giacomini-White
Wald test, not the `gw_test` alias), and the V0 inventory shows this
callable has only a directional DGP oracle and no exact anchor, this is a
genuine open calibration issue for multi-horizon use, worth a code fix
(e.g. a Kiefer-Vogelsang-style fixed-b adjustment or an HLN-type scaling)
rather than a docs note.

### Finding 3 — `model_confidence_set` under-covers the equal-losers global null (0.82 vs ≥0.90)

P(all 5 tied models retained) = .818 [.785,.848] against the 1−α=.90
floor. Confirmed stable across a pre-registered sweep: n_boot 1000→2000
(.833→.820 — rules out bootstrap MC noise), n=100→200 (.833→.850 — not
vanishing with data), common factor removed (.857), α=.05 (.867 vs .95
target — relative shortfall even larger). **Diagnosis: the first-step
elimination test built on `_mcs_statistic` (the exact private helper WP-V0
flagged as the single biggest untested gap in tests.py) over-rejects the
"no model is worse" global null by a stable ~8pp margin.** By contrast the
single-dominant-model design PASSES its one-sided coverage floor (.947 ≥
.90), so the sequential elimination logic downstream of the statistic
behaves sanely; the distortion is specifically in the bootstrapped
max/range statistic's null calibration. This matches the known behavior of
Tmax-type MCS bootstraps being liberal in small K·n panels, but the margin
here is large enough that it should be treated as an open item on
`_mcs_statistic` (V0 priority #1), not shrugged off.

### Finding 4 — `pesaran_timmermann_test` mildly oversized at n=50

Re-confirmed at R=15,000: .0559 [.0512,.0609] @α=.05, .1084 [.1020,.1151]
@α=.10 — small but real. n=200 is cleanly in band on the identical code
path. **Diagnosis: large-n asymptotic-normal reference of the PT (1992)
statistic; finite-sample discreteness at n=50.** The formula itself is
parity-anchored (tstests/rugarch, WP-V0). Practical impact minor
(~0.6–0.8pp at n=50); referees rarely run PT on 50 observations, but it's
now measured rather than assumed.

### Finding 5 — `superior_predictive_ability_test` oversized under serially-correlated losses

The iid design is in band (.0633 [.0482,.0813] @α=.05), but ANY serial
correlation in the loss panel (AR(1) idio ρ=0.3 + common factor) pushes
rejection to ~1.5–2x nominal, and a pre-registered sweep shows it is
insensitive to every user-facing knob: p_value_type
consistent/upper/lower (.096/.074/.091), studentize off (.093), block
length 1/5/10/20/auto (auto resolves to ~3; 10 → .101, 20 → .131), and
n=100/250/500 (.088/.074/.089 — does not shrink with data). **Diagnosis:
the distortion tracks the serial-correlation handling of the
`arch.bootstrap.SPA` backend path (block-bootstrap calibration of the
studentized max statistic), not the panel-orchestration layer in
macroforecast (which the iid design exercises identically and passes).**
Downstream implication: SPA p-values on realistic (serially-correlated)
loss panels are anti-conservative; MCS finding 3 above is the same
qualitative direction, so paper tables built on either should treat
borderline rejections at α with caution. Documented precision limit: at
R=1500 with a bootstrap test, the 99% band is ±~1.5pp; the observed .090
vs .050 excess is ~5x that band, so the verdict is not budget-limited.

## What passed cleanly (size behavior verified, new positive anchors)

- `dm_test` at h=1 (both n, both corrections, both α) and at n=200 h=4 —
  the HAC/df/HLN machinery is correctly calibrated whenever the effective
  sample is adequate.
- The HLN-direction property itself (correction moves size toward nominal
  at small n, large h) — asserted, not just narrated.
- `clark_west_test` under the nested null — does NOT over-reject despite
  demonstrably induced nested bias (mean loss differential negative in the
  same replications; the unadjusted statistic rejects at only .001 while
  CW-adjusted sits at .027 — the adjustment works in the documented
  direction), and shows healthy mid-range power (.55) against a small true
  alternative. Consistent with Clark-West (2007)'s "approximately
  correctly sized, slightly conservative one-sided test."
- `model_confidence_set` retains a genuinely (slightly) dominant model at
  .947 ≥ 1−α — the headline MCS guarantee holds in the dominant case.
- `pesaran_timmermann_test` at n=200; `giacomini_white_test` at h=1;
  `superior_predictive_ability_test` on iid losses.

## Engineering notes

- New marker `mc` registered in `pyproject.toml`; CI default run updated
  to `-m 'not deep and not slow and not mc'` (verified: `pytest tests/mc/
  -m 'not deep and not slow and not mc'` collects 0 of 21). Every test
  carries an individual `@pytest.mark.timeout`.
- All seven xfails are `strict=True`: if a future code fix repairs any
  distortion, the suite FAILS loudly (XPASS) and forces the finding to be
  re-classified — the xfails are tripwires, not mufflers.
- DM/GW share one DGP helper (MA(h−1) overlapping-error construction — the
  textbook h-step forecast-error autocovariance) but are separate suites:
  different reference distributions (t vs chi²) and different HAC
  constructions, so one passing cannot mask the other.
- Diagnostic sweep scripts were run out-of-tree (`~/scratch_wpv3/` on
  server1) and their numbers are transcribed above; the in-tree suite
  contains only the decisive design cells to protect the runtime budget.
