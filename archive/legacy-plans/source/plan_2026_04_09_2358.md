**tree path package**로 가려면, 진짜로 **1.1, 1.2, 2.1.1, 2.1.2** 식으로 내려가야 합니다. 그래야 “어떤 forecasting paper든 우리 package의 one path”가 됩니다.

현재 `macrocast`는 설계 문서 기준으로 `data / pipeline / evaluation` 3-layer 뼈대를 갖고 있고, 코드 기준으로는 `ForecastExperiment`의 direct multi-horizon pseudo-OOS, `FeatureBuilder`, `metrics`, `MCS`가 이미 있습니다. 따라서 아래 설계는 **현재 구조를 버리는 것이 아니라**, 그 위에 **상위 tree-path taxonomy**를 얹는 방식이 가장 자연스럽습니다.

핵심 원칙은 딱 네 가지입니다.

1. **one path = one fully specified forecasting study**
2. **enumerated choice**와 **numeric/free parameter**를 분리
3. **fixed axis**와 **sweep axis**를 분리
4. Python module tree와 experiment tree를 분리

즉, 실제 package module은 단순하게 두고,
**`taxonomy/ + registries/ + recipes/ + runs/`** 구조로 가는 것이 맞습니다.

---

## A. 추천 top-level tree

```text
macrocast/
├── data/                     # data engine
├── pipeline/                 # training engine
├── evaluation/               # metrics/reporting engine
├── tests_eval/               # statistical tests engine
├── importance/               # importance / interpretability engine
├── taxonomy/                 # all selectable tree-path registries
│   ├── 0_meta/
│   ├── 1_data/
│   ├── 2_target_x/
│   ├── 3_preprocess/
│   ├── 4_training/
│   ├── 5_evaluation/
│   ├── 6_stat_tests/
│   └── 7_importance/
├── recipes/                  # named studies / paper replications
│   ├── papers/
│   ├── baselines/
│   ├── benchmarks/
│   └── ablations/
├── runs/                     # realized experiment outputs
└── cli.py
```

그리고 실제 **canonical path string**은 예를 들어 이렇게 갑니다.

```text
/domain=macro
/data=fred_md
/freq=monthly
/info_set=revised
/sample=1959_01-2024_12
/oos=1990_01-2024_12
/task=point_direct
/h=1,3,6,12
/target=INDPRO
/x_map=all_minus_target
/target_prep=logdiff
/x_prep=em+outlier_to_missing+zscore
/features=pca8+ylag12
/framework=expanding
/validation=blocked_kfold5
/model=random_forest
/tuning=random_search_100
/metric=oos_r2
/benchmark=ar_bic
/stat=dm_hln+mcs
/importance=shap_grouped
/output=parquet+latex+plots
```

# C. 실제 path system은 어떻게 저장할 것인가

이건 매우 중요합니다.
**모든 선택지를 실제 filesystem path segment로 다 박으면 망합니다.** 너무 길고, 숫자 파라미터가 path를 오염시킵니다.

그래서 분리해야 합니다.

## 1. enum choice는 path segment

예:

```text
/data=fred_md/info_set=revised/framework=expanding/model=rf
```

## 2. numeric/free parameter는 leaf config

예:

```yaml
horizons: [1,3,6,12]
sample_start: "1959-01"
sample_end: "2024-12"
n_factors: 8
n_lags: 12
max_depth_grid: [3,5,7]
```

## 3. 추천 저장 구조

```text
recipes/
  papers/
    clss_2022.yaml
    data_transformations_matter.yaml
    fred_md_baseline.yaml

runs/
  macro/
    fred_md/
      revised/
        target_INDPRO/
          recipe_clss_2022/
            run_0001/
              config.yaml
              predictions.parquet
              metrics.parquet
              tests.json
              importance.parquet
```
